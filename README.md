# CityPulse AI

AI-платформа мониторинга городской среды Алматы, объединяющая транспорт, экологию, карту города, персональный экологический след и AI-ассистента в едином интерфейсе.

## Возможности

- **Транспортный мониторинг** — средняя скорость, загруженность дорог, инциденты по районам
- **Экологический мониторинг** — AQI, PM2.5, PM10 по районам
- **Комбинированный анализ** — выявление связи между транспортной нагрузкой и качеством воздуха
- **Карта города** — полигоны районов Алматы, окрашенные по risk score, с точками проблем
- **AI-сервис (OpenAI)** — генерация выводов, рекомендаций и ответов на вопросы; fallback без API
- **AI-ассистент** — чат для вопросов о городе, возможностях платформы, прогулках и планах
- **Личный экологический калькулятор** — Air Impact Score по параметрам автомобиля
- **Здоровье и прогулки** — рейтинг районов для прогулок, рекомендации для групп риска
- **Пробки и маршруты** — проблемные участки, альтернативные маршруты, интеграция с 2GIS
- **Карта переработки** — пункты приема вторсырья (demo-данные)

## Технологии

- Python 3.10+
- Streamlit
- Pandas
- Plotly
- Pydeck
- OpenAI API (опционально)

## Установка и запуск

```bash
# Клонировать репозиторий
git clone <repo-url>
cd citypulse-ai

# Создать виртуальное окружение
python3 -m venv venv
source venv/bin/activate

# Установить зависимости
pip install -r requirements.txt

# (Опционально) Настроить OpenAI API
cp .env.example .env
# Отредактировать .env и вставить свой OPENAI_API_KEY

# Запустить приложение
streamlit run app.py
```

Приложение работает и без OpenAI API — в этом случае используется rule-based fallback.

## AI-сервис

Модуль `services/ai_service.py` — единая точка AI-логики:

- **С OpenAI API** — генерирует human-friendly выводы, рекомендации, ответы ассистента, интерпретацию эко-индекса
- **Без API (fallback)** — rule-based логика, приложение полностью функционально

Настройка через `.env`:
```
OPENAI_API_KEY=sk-...
OPENAI_MODEL=gpt-4o-mini
```

## Логика определения проблем

### Транспорт

| Показатель | Норма | Внимание | Критично |
|---|---|---|---|
| Загруженность | < 50% | 50–80% | >= 80% |
| Средняя скорость | > 25 км/ч | — | < 25 км/ч (усиливает) |
| Инциденты | < 3 | — | >= 3 (усиливает) |

### Экология

| Показатель | Норма | Внимание | Критично |
|---|---|---|---|
| AQI | <= 50 | 51–100 | > 100 |
| PM2.5 / PM10 | — | — | Высокие значения усиливают приоритет |

### Комбинированная проблема

Если в одном районе одновременно загруженность >= 80% и AQI > 100 — фиксируется как **комбинированная проблема высокого приоритета**.

## Данные

В MVP используются mock-данные для 5 районов (Бостандыкский, Алмалинский, Медеуский, Ауэзовский, Наурызбайский) за 2 дня с тремя временными точками (утро, день, вечер). В данных заложены критические ситуации в вечерние часы для демонстрации работы системы.
